# Contributing

Open an issue in the [OpenCorex OpenRevenue repository](https://github.com/opencorex-org/openrevenue/issues) before substantial design work. Keep bounded contexts isolated, add forward-only migrations, preserve rule/form versions, use minor units for money, and add tests. Run `make format lint test`. Commits should follow Conventional Commits. Contributions require the Developer Certificate of Origin (`Signed-off-by`). See the architecture and development guides before submitting a pull request.
